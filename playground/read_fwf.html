readFwf — Fixed-Width Format I/O

readFwf() mirrors pandas read_fwf(). Parse text where each column occupies a fixed range of character positions. Column boundaries can be auto-inferred from whitespace, specified as colspecs ([start, end] pairs), or provided as column widths.

1 — Auto-infer column boundaries

Pass a text string to readFwf(). The first row is the header. Column boundaries are inferred automatically from where content appears. 

const text = [
  "Name       Age  Score",
  "Alice       29   88.5",
  "Bob         34   91.0",
  "Charlie     22   79.3",
].join("\n");

const df = tsb.readFwf(text);
console.log("shape:", df.shape);
console.log("columns:", [...df.columns.values]);
console.log("Name  dtype:", df.col("Name").dtype.name);
console.log("Age   dtype:", df.col("Age").dtype.name);
console.log("Score dtype:", df.col("Score").dtype.name);
console.log("Names:", [...df.col("Name").values]);
console.log("Ages :", [...df.col("Age").values]);


    


    

      
2 — Explicit colspecs

      

        Provide [start, end] pairs (end-exclusive, matching pandas) for
        precise control over column extraction.  Useful when columns touch without
        whitespace gaps.
      

      


const text = [
  "Alice 29 88.5",
  "Bob   34 91.0",
].join("\n");

const df = tsb.readFwf(text, {
  colspecs: [[0, 6], [6, 9], [9, 13]],
  header: null,
  names: ["Name", "Age", "Score"],
});
console.log("Names:", [...df.col("Name").values]);
console.log("Ages :", [...df.col("Age").values]);
console.log("Scores:", [...df.col("Score").values]);


      

      
      

    


    

      
3 — Column widths

      

        If every column has a fixed width, supply a widths array.
        The boundaries are computed automatically from the cumulative widths.
      

      


const text = [
  "ID  Name     Score",
  "1   Alice      88",
  "2   Bob        91",
  "3   Charlie    79",
].join("\n");

const df = tsb.readFwf(text, { widths: [4, 9, 5] });
console.log("columns:", [...df.columns.values]);
console.log("IDs   :", [...df.col("ID").values]);
console.log("Names :", [...df.col("Name").values]);
console.log("Scores:", [...df.col("Score").values]);


      

      
      

    


    

      
4 — Comment lines and missing values

      

        Lines starting with comment are ignored.  Cells containing
        "NA", "", or custom strings (via naValues)
        become null.
      

      


const text = [
  "# Generated 2024-01-01",
  "Name       Value",
  "# unit: meters",
  "Alice         42",
  "Bob           NA",
  "Charlie     -999",
].join("\n");

const df = tsb.readFwf(text, {
  comment: "#",
  naValues: ["-999"],
});
console.log("Names :", [...df.col("Name").values]);
console.log("Values:", [...df.col("Value").values]);


      

      
      

    


    

      
5 — skipRows, nRows, and indexCol

      

        skipRows skips data rows after the header.  nRows
        caps the number of rows read.  indexCol promotes a column to the
        row index.
      

      


const text = [
  "id   name     score",
  "1    Alice       88",
  "2    Bob         91",
  "3    Charlie     79",
  "4    Diana       95",
].join("\n");

const df = tsb.readFwf(text, { indexCol: "id", nRows: 3 });
console.log("shape  :", df.shape);
console.log("columns:", [...df.columns.values]);
console.log("index  :", [...df.index.values]);
console.log("scores :", [...df.col("score").values]);
